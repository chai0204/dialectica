### project_document.mdの残る議論の余地について

1.1. 命題の粒度制御についてはこれは完全に参加者に任せていいと思います。仮にある命題の分解を考えてみましょう。
```
初期の命題→消費税は下げるべきだ。
これを真と考える参加者は「消費税は経済に悪影響を与えている」という命題が真であることを前提としているかもしれません。一方で、そうは考えない人はその前提が偽であると考えているかもしれません。その場合は、「消費税は下げるべきだ。」という命題を議論する前に明らかにその前提を構成している「消費税は経済に悪影響を与えている」という命題の議論が必要です。
そして、「消費税は経済に悪影響を与えている」という命題を真だと考える人はさらに「消費税が下がれば家計の可処分所得は増える」と「可処分所得が増えれば家計消費は増加する」、「家計消費が増大すれば可処分所得は増える」といった各命題を真だと考え、それらから「消費税は経済に悪影響を与えている」が真である前提としていたとすると、その前提の脆弱性として「可処分所得が増えれば家計消費は増加する」が偽であり、可処分所得は消費と貯蓄の2つの経路があるため、貯蓄が増えるばかりで必ずしも消費が増えるとは限らないといった反駁ができたりすると思います。
```
このように命題をどこまで分解・あるいは前提をさかのぼるかは1人でやるには大変ですが、大衆というマンパワーあるいはAIを使うことで、可能な限りの深堀をする形になると思います。一方で、初期の命題についてはじめっから前提や根拠を明らかにしない主張だけであってもよいと思います。そのいずれにしても議論の余地が残っており、議論されているという点で同じであり、議論の深さは些細な問題でどれだけ議論を深化させるかは参加者次第です。

1.2. 関手は特別実装しなくてよいと思います。まず、異なるドメイン間の構造的対応はそれぞれの議論が進むにつれてノードとエッジは変化する一方で完全一致というのはほぼないかあっても簡単に解除されます。ただ、実際にその構造的対応の視点があること自体は良いと思います。それも参加者の解釈という他と同じレベルで行えればよいと思います。そして参加者の解釈は各命題や命題同士の関係、他参加者の解釈という単一を対象とするに限らず、それらを包含したある領域も対象とできるようにすれば参加者による構造的な類似の解釈も述べやすいと思います。

1.5. 解釈と命題の区別は確かに問題です。可能か限りすべてを命題として扱ったほうがきれいだとは思いますが、どうでしょうか。あなたの意見を聞きたいです。

2.3. 初期は汎用モデルで始めます。

3.1. システムについてはオープンソースにしてそのうえで、それをもとにシステム自体も議論の対象とするのがよいと思います。そして、管理者がプラットフォーム上での議論の状況に応じて理由を明記したうえで変更したりPRをマージするのがよいと思います。また、githubの各ＰＲなどのコメントでその時点での結論の詳細な導出について記述するので良いと思います。

3.2. 承認は存在しなくてよいと思います。その代わり命題同士のリレーションで同一を用意すればよいと思います。また、そのリレーションも同一があれあ類似もあるといった量子性を許容すればよいと思います。

3.3. AIと人間は等しく参加者として区別しなくてよいと思います。このプラットフォーム上ではそれらは完全に平等であり、かつ数が多いことによってある命題の解釈が偏ることを防止するためです。

4.2. 命題に些末も本質も評価はしません。仮に「水は100度で沸騰する」も気圧によってはその限りではなく今でこそ当たり前ですが、もし紀元前の文明であればそんな当たり前も検証して確かめてきたはずです。このプラットフォームは人類の総合知であり、すべては人類の発見です。また、命題の前提を究極的にたどるとどこかですべての人の解釈が一致するレベルまで自明な命題まで行きつくことも歓迎すべきだと思うので、そのような命題が乱立するのも決して悪いことではないと考えます。

5.1. 学術の世界にも現状のコンセンサスと主流でない仮説などが存在することは自然科学でもよくあります。確かに究極的には完全な事実が確認できるまではすべての可能性を考慮しランク付けのようなことはあまり適切ではないと考えられるかもしれませんが、そこは人間の認知の面を考慮するとある程度情報を軽量化する必要があるため妥協点であると考えられるかもしれません。

5.2. 判定を行わないというと確かに問題となるかもしれませんね。ただ、ここでの本質はシステムは常に統計学的な視点をとることで客観性を担保するということが重要なのだと思います。個別の判断はさまざまなバイアスにその多くの説明力が宿りますが、統計的・マクロ的な視点だとある程度そのバイアスやノイズによる説明力ではない本質的な説明力を抽出できると仮定しています。これも論じるべき内容かもしれませんが、このプロジェクトの設計も同様すべてのことは永遠に議論できますが、永遠に議論するのではなくどこかで区切りをつけなければいけないと思います。そうしないとなにもできないから。ただ、私はせめてその議論の未熟性を正確に把握しておくことが大事なのだと思います。

5.3. このプラットフォームは確かに初期は限られた人だけが参加し途中参加者はその初期参加者の積み上げたものの上で参加することを余儀なくされますが、それは許容していいものだと思います。人類は1世代ではせいぜい100年しか生きられず、その中で探求できることはたかが知れていますが、人類が現代ここまで科学技術を発展させたのは先人の知の積み重ねのおかげだと思います。このプラットフォームはその今までの人類の知の積み重ねを再集計するプロセスとも解釈でき、そのような積み重ねという人類の性質はもはや本質でありその機構を再現できている点は歓迎すべき点だと考えられます。
